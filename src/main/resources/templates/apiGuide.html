API FUNCIONANDO CORRECTAMENTE

Endpoints:

Productos

  • GET "/productos/traer": devuelve el total de productos en la base de datos
  • POST "/productos/crear": permite crear un producto a la vez
  • POST "/productos/batch": permite crear productos en base a un array
  • DELETE "productos/borrar/{id}": permite eliminar un producto indicando su ID
  • PUT "productos/editar/{id}": permite editar un producto indicando su ID
  • PUT "productos/actualizar/stock": permite actualizar el stock de todos los productos
  • PUT "productos/actualizar/precio": permite actualizar los precios de todos los productos

Publicaciones Mercadolibre

  • POST "/ml/crear": permite crear una publicación a la vez
  • GET "/ml/traer": devuelve el total de publicaciones en la base de datos
  • POST "/web/estructura": Devuelve un array con la estructura del modelo de ML
  • DELETE "/ml/borrar/{id}": permite eliminar una publicación indicando su ID
  • PUT "/ml/editar/{id}": permite editar una publicación indicando su ID
  • PUT "/ml/editarParcial/{id}": Permite editar parcialmente una publicación. Autodetecta campos

Publicaciones página web

  • POST "/web/crear": permite crear una publicación a la vez
  • GET "/web/traer": devuelve el total de publicaciones en la base de datos
  • POST "/web/batch": permite crear publicaciones en base a un array
  • DELETE "/web/borrar/{id}": permite eliminar una publicación indicando su ID
  • PUT "/web/editar/{id}": permite editar una publicación indicando su ID
  • PUT "web/actualizar/stock": Toma un body y actualiza el stock de las publicaciones en consecuencia
  • PUT "web/actualizar/precio": Toma un body y actualiza el precio de las publicaciones en consecuencia
  • PUT "web/actualizar/batch": Toma un body y actualiza todas las publicaciones en consecuencia

Reputación

  • GET "reputacion/datos": Trae todos los datos de reputación
  • PUT "reputacion/editar": Permite editar los primeros tres valores del objeto principal
  • POST "reputacion/reclamos": Toma un array y sobrescribe el array de reclamos en el objeto principal
  • POST "reputacion/mediaciones": Toma un array y sobrescribe el array de mediaciones en el objeto principal
  • POST "reputacion/cancelaciones": Toma un array y sobrescribe el array de cancelaciones en el objeto principal
  • POST "reputacion/demoras": Toma un array y sobrescribe el array de demoras en el objeto principal
